## Скиллы программиста (Гейл)

| Скилл | Когда использовать |
|-------|--------------------|
| **test-runner** | Запуск тестов, парсинг результатов, написание тестов. "Запусти тесты", "coverage" |
| **code-review** | Ревью кода, поиск антипаттернов, оценка diff. "Посмотри код", "review PR" |
| **dep-audit** | Аудит зависимостей: уязвимости, лицензии, обновления. "npm audit", "что устарело" |
| **git-wizard** | Сложные git-операции: rebase, cherry-pick, конфликты, восстановление |
| **perf-profiler** | Профилирование: Lighthouse, Node.js, SQL. "Тормозит", "утечка памяти" |
| **content-pipeline** | Автоконвейер контента: Туко→Лало→Ким. "Напиши пост", "дайджест", "контент для канала" |

## Decision Tree - какой скилл когда

```
Задача получена → определи тип:

├─ "Не работает / баг / ошибка / сломалось"
│  └─ systematic-debugging (ВСЕГДА первый)
│     ├─ Баг в тестах? → test-runner
│     ├─ Баг из-за зависимости? → dep-audit
│     ├─ Баг в производительности? → perf-profiler
│     └─ Баг архитектурный? → code-review
│
├─ "Напиши код / сделай фичу / реализуй"
│  ├─ Сложная задача? → writing-plans (сначала план)
│  ├─ Нужен ресёрч технологии? → deep-research-pro
│  ├─ Несколько вариантов? → brainstorming
│  └─ После реализации → test-runner → code-review → quality-check
│
├─ "Проверь / ревью / оцени код"
│  ├─ Конкретный diff/PR? → code-review
│  ├─ Перед публикацией? → quality-check
│  └─ Нужны тесты? → test-runner
│
├─ "Тормозит / медленно / оптимизируй"
│  └─ perf-profiler
│     ├─ Проблема в зависимостях? → dep-audit
│     └─ Проблема в коде? → code-review
│
├─ "Git / rebase / конфликт / откати"
│  └─ git-wizard
│
├─ "Зависимости / npm audit / обнови пакеты"
│  └─ dep-audit
│
├─ "Запусти тесты / coverage / тесты падают"
│  └─ test-runner
│     └─ Падают непонятно? → systematic-debugging
│
├─ "Проверь сайт / SEO / аудит"
│  └─ audit-website + perf-profiler (если перформанс)
│
├─ "Напиши пост / дайджест / контент для канала"
│  └─ content-pipeline (автоконвейер: Туко→Лало→Ким)
│
└─ "Схема / диаграмма архитектуры"
   └─ excalidraw
```

## Цепочки скиллов (workflows)

**Новая фича (полный цикл):**
`brainstorming → writing-plans → [код] → test-runner → code-review → quality-check`

**Баг-фикс:**
`systematic-debugging → [фикс] → test-runner → quality-check`

**Перед релизом:**
`dep-audit → test-runner → code-review → quality-check`

**Оптимизация:**
`perf-profiler → [измерение] → code-review → [фикс] → perf-profiler → [сравнение]`

**Контент для канала:**
`content-pipeline: Туко(ресёрч) → Лало(копирайтинг) → Ким(ОТК) → [доработка] → финал`
